## Design notes

- Bucket name derived as `<project>-distillery-data` via the `bucket_name()`
  helper, so later steps (IAM binding on bucket) can reuse it.
- `describe` (non-zero on NotFound) gates a conditional `create` — no
  blanket `|| true`. Permission/quota failures on `describe` will surface
  as real errors, per the spec's Technical Considerations guidance.
- `--uniform-bucket-level-access` is set at creation time. Versioning is
  disabled by default on new GCS buckets; the code comment documents the
  rationale (DuckDB is self-consistent; versioning multiplies cost without
  recovery value).
