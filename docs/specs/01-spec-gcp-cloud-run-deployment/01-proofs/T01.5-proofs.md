## Design notes

- **Attribute condition** (`attribute.repository == '<owner>/<name>'`) is
  set at provider creation time. This is the single most important
  security control on the WIF path — without it, any GitHub repo could
  mint tokens for the deployer SA.
- **principalSet path** uses the project *number* (not ID), looked up
  via `gcloud projects describe`. The GCP_WIF_PROVIDER repo variable
  emitted by the summary uses the same number.
- **Summary to stdout**: `log()` writes to stderr, so the summary block
  is the only thing on stdout and can be piped cleanly
  (`bash gcp/bootstrap.sh ... | tail -n 20 | pbcopy`).
- **Attribute mapping** includes `actor` and `ref` even though the
  current condition only uses `repository` — preserves the option to
  tighten later (e.g. `attribute.ref == 'refs/heads/main'`) without
  having to recreate the provider.
- **`roles/iam.workloadIdentityUser`** is bound SA-scoped (on the
  deployer SA resource), not project-scoped. Only the deployer SA is
  impersonable via this principalSet.
